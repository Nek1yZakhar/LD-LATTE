## 1. Назначение и цель тикета

Цель **TICKET-10E** — наполнение одностраничного веб-сайта **LD Latte Demo UI** высококачественными, 100% доказательными интерактивными модулями (Interactive Evidence Views). 

Данный этап преобразует каркас Части 1 сайта из статического shell-представления в полноценную **рабочую среду доказательств (Proof Studio)**, позволяющую проверяющей стороне (evaluators / экспертам LD Latte) глубоко проинспектировать:
1. Архитектурный граф пайплайна и контракты данных Pydantic по каждому из 8 этапов.
2. Синтез идеального профиля инфлюенсера (`ideal_portrait.json`) и обоснование алгоритма.
3. Воронку фильтрации кандидатов (переключение между Топ-5 шорт-листом и 17 активными профилями).
4. Детализацию математического скоринга: векторное сходство Qwen3-Embedding-0.6B, пофичевые оценки (`niche`, `ER`, `recency`, `sponsorship`, `language`) и сигмоидную нормализацию логитов BAAI/bge-reranker-v2-m3.
5. Роль VLM Visual Sanity Pass (`Qwen2.5-VL`) с эстетическими вердиктами и фактологическими заземлениями (`grounding_facts`).
6. Генератор персонализированных PR-офферов с контролем anti-robotic QA и инспектором системного промпта `prompts/outreach_offer.md`.

## 2. Разработанные доказательные компоненты

Все новые модули размещены в изолированной папке `app/src/components/evidence/`:

### 2.1 `PipelineEvidenceGraph.tsx`
* **Интерактивность**:
  * Сетка карточек 8 этапов пайплайна с индикацией статуса, файлов Python (`src/ingest/clean.py`, `src/fetchers/enrich.py`, `src/analyzers/portrait.py`, `src/search/discover.py`, `src/scoring/embed.py`, `src/scoring/rerank.py`, `src/scoring/vlm_sanity.py`, `src/outreach/generator.py`).
  * Клик на любой этап открывает **Stage Inspector Modal** с отображением входного и выходного контрактов данных, описания работы и Pydantic-гарантий.
  * Выделенная плашка VLM-узла (Stage 07), явно подчеркивающая архитектурное ограничение: VLM используется **только для 5 топов шорт-листа**, а не для первичного поиска.
  * **Ideal Blogger Profile Inspector**: Модальный инспектор эталонного профиля с выводом параметров `IdealBloggerProfile` и оригинального обоснования Llama-3.3-70B (`rationale`).

### 2.2 `CandidateEvidenceStudio.tsx`
* **Интерактивность**:
  * Переключатель режимов просмотра: **«🔥 Топ Шорт-лист (5)»** vs **«📊 Все Кандидаты (17)»**.
  * Живой текстовый фильтр (поиск по имени, хэндлу `@username` или нише).
  * Карточки кандидатов с прогресс-баром `Composite Score`, плашками `VLM Pass`, `Shortlist` и фактологическими метриками (подписчики, ER, ниша, язык).
  * **Candidate Deep-Dive Modal**:
    * **Вкладка 1. Математический скоринг**: вывод 5 пофичевых оценок (`niche_match`, `er_match`, `recency_match`, `sponsorship_match`, `language_match`), весов фич, сырых логитов и нормированных оценок BGE-Reranker.
    * **Вкладка 2. VLM Audit & Facts**: результаты вердикта Qwen2.5-VL (`vlm_sanity_passed`, `vlm_aesthetic_notes`) и нумерованный список `grounding_facts`.
    * **Вкладка 3. Персональный Оффер**: просмотр сгенерированного коммерческого предложения прямо в карточке блогера.

### 2.3 `OutreachOfferStudio.tsx`
* **Интерактивность**:
  * Переключатель между сгенерированными офферами для блогеров шорт-листа (`@jd_cosm`, `@dddinaaaaaa`, `@llaurraiiam`, `@bazhenova_alenaa`, `@mishandkatya`).
  * Отображение темы письма (Subject), форматированного тела предложения (Body) и тегов персонализации (`personalized_elements`, `grounding_facts`).
  * Кнопка быстрый копирования текста в буфер обмена.
  * **Prompt Inspector Modal**: Инспектор промпта `prompts/outreach_offer.md` с демонстрацией системных правил, фильтра кириллицы (`cyrillic_count >= 15`), экстракции имен и предохранителей anti-robotic QA.

## 3. Соблюдение ключевых ограничений (Guarantees)

1. **Strict Zero-Mock Data Policy**: Все данные в UI поступают исключительно из центрального слоя данных `app/src/data/` (`content/pipeline_data.ts` & `adapters/index.ts`), объединяющего 100% реальные JSON/MD артефакты пайплайна.
2. **Изоляция слоя представления**: frontend не содержит дублирования Python-логики скоринга или backend-запросов; все расчёты отображаются строго по зафиксированным артефактам.
3. **Сохранение фрейминга**: В интерфейсе четко акцентируется позиционирование системы как **modular AI pipeline (agent-ready)** с разделением ответственности между локальными эмбеддингами, кросс-энкодером и VLM-аудитом.
4. **Адаптивность и производительность**: Компоненты оптимизированы для корректного отображения как на десктопных мониторах, так и на мобильных устройствах. Успешно пройден со сборкой `npm run build`.

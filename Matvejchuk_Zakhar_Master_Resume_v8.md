**Approval Service — асинхронный микросервис для фриланс-заказчика (NDA) | 2026**

Backend/Python Developer (freelance, solo)

Спроектировал и реализовал по заказу клиента (NDA) асинхронный микросервис управления workflow согласования контента. Довёл проект от рабочего MVP до production-grade стандартов: устранил уязвимости, добавил CI/CD и полную архитектурную документацию.

**Что сделал:**

- Спроектировал сервис на FastAPI (async) с чистым разделением на слои: routes → services → repositories → models, реализовал стейт-машину переходов заявки (pending → approved/rejected/cancelled) с запретом переходов из финальных состояний (HTTP 409).
- Реализовал паттерн Transactional Outbox: события изменений атомарно записываются в таблицу outbox_events в той же транзакции, что и изменение статуса заявки, гарантируя доставку at-least-once во внешние системы.
- Внедрил идемпотентность создания заявок через Idempotency-Key и UniqueConstraint на уровне БД, исключив дублирование запросов.
- Провёл самостоятельный security-аудит и устранил критические уязвимости: заменил фиктивную Base64-авторизацию на полноценный JWT (HS256) с проверкой подписи и сроком действия, добавил проверку принадлежности пользователя к списку reviewer_user_ids перед approve/reject, убрал небезопасное дефолтное значение SECRET_KEY.
- Настроил CI/CD-пайплайн (GitHub Actions): автоматический линт (ruff), проверка типов (mypy) и прогон тестов с покрытием (pytest-cov) на каждый push/PR.
- Задокументировал архитектуру в DESIGN.md: модель данных, диаграмма компонентов в Mermaid, идемпотентность, границы сервиса, известные trade-offs — по той же методологии ADR, что применяется в OSINT-агрегаторе МИБ.
- Покрыл ключевые сценарии 12 интеграционными тестами (pytest-asyncio): изоляция воркспейсов, идемпотентность, обработка ошибок переходов состояний, авторизация.
- Задеплоил через Docker Compose (PostgreSQL + приложение), с миграциями через Alembic.

# Методология разработки

Использую AI-native подход к инженерии по циклу Plan → Approve → Execute → Audit: проектирую архитектуру системы, выбираю стек, декомпозирую проект на тикеты и формирую state/agent rules — делегируя генерацию кода ИИ-агенту (Antigravity IDE) под полным контролем: Planning Mode перед кодом, accept/reject каждого изменения на уровне diff — делегируя генерацию кода ИИ-агентам с полным техническим контролем результата. Перед каждым проектом — исследование существующих решений и архитектурных паттернов.
